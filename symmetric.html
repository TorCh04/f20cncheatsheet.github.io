Network Security: Symmetric Cryptography
Exam Revision Sheet

1. Modern Cryptographic Algorithms

2. Types of Ciphers

  • Symmetric ciphers: Same key k for encryption & decryption. Key must be exchanged securely.
  • Two main types:
    • Block ciphers: Encrypts fixed-size blocks of plaintext.
    • Stream ciphers: Encrypts plaintext bit-by-bit using a keystream.

2.1 Block Ciphers

  • Plaintext split into blocks: m₁, m₂, ..., mₙ
  • Each block encrypted with same key k in a specific mode.

2.2 Stream Ciphers

  • Key k generates a keystream: k₁, k₂, ...
  • Each plaintext bit mᵢ encrypted with kᵢ (usually XOR).
  • RC4: Outdated, insecure, used in early WiFi.
  • ChaCha20: Modern, used in VPNs, TLS 1.3.

3. Designing Block Ciphers

  • Block cipher: maps n-bit plaintext to n-bit ciphertext.
  • Number of possible mappings: (2ⁿ)!
  • Each mapping = a possible key.

4. Bits, Bytes, and Blocks

  • Bit: 0 or 1
  • Byte: 8 bits
  • Block size: typically 64–256 bits
  • Number of possible blocks: 2ⁿ
  • Key size: typically 64–256 bits
Tip: 2¹⁰ ≈ 10³, 2²⁰ ≈ 10⁶, 2⁶⁰ ≈ 10¹⁸

5. Good vs. Bad Cipher Design

  • Linear mappings (e.g., c = a·m) are weak: easy to break with known plaintext.
  • Non-linear mappings are stronger, especially with high-degree polynomials and key-dependent coefficients.
  • Modern ciphers use:
    • Confusion: Obscure key-ciphertext relationship (via substitution).
    • Diffusion: Spread plaintext bits across ciphertext (via transposition).
  • Multiple rounds of substitution & transposition improve security.

6. Data Encryption Standard (DES)

  • Block cipher, 64-bit blocks, 56-bit key (+8 parity bits).
  • 16 rounds, Feistel structure: splits block into L and R halves.
  • Encryption round:
    Lᵢ = Rᵢ₋₁
    Rᵢ = Lᵢ₋₁ ⊕ f(Kᵢ, Rᵢ₋₁)
  • Each round uses a 48-bit subkey from the main key.
  • DES is now considered insecure (key too short).

7. Advanced Encryption Standard (AES/Rijndael)

  • Block cipher, 128-bit blocks, key sizes: 128/192/256 bits.
  • 10, 12, or 14 rounds (depending on key size).
  • Each round:
    • SubBytes: Substitute each byte.
    • ShiftRows: Permute rows.
    • MixColumns: Matrix multiplication (diffusion).
    • AddRoundKey: XOR with round key.
  • All steps invertible for decryption.
  • Attacks: Only practical attacks are side-channel (timing) attacks. Use constant-time implementations to defend.

8. Related Ciphers

  • IDEA: 64-bit blocks, 128-bit key, efficient, used in PGP.

9. Modes of Operation

  • How block ciphers encrypt multiple blocks.
  • Main modes:
    • ECB (Electronic Codebook): Each block encrypted independently.
      cⱼ = Eₖ(mⱼ)
    • CBC (Cipher Block Chaining): Each block XORed with previous ciphertext before encryption.
      cⱼ = Eₖ(mⱼ ⊕ cⱼ₋₁)
    • CTR (Counter): Block cipher used as stream cipher, keystream from counter.
      cⱼ = Eₖ(iv + j - 1) ⊕ mⱼ
    • CFB (Cipher Feedback): Previous ciphertext encrypted, then XORed with plaintext.
      cⱼ = Eₖ(cⱼ₋₁) ⊕ mⱼ
    • OFB (Output Feedback): Output of cipher fed back to generate keystream.
      Oⱼ = Eₖ(Oⱼ₋₁), cⱼ = Oⱼ ⊕ mⱼ
Exam Tips:
  • Understand the difference between block and stream ciphers.
  • Be able to explain and compare ECB, CBC, CTR, CFB, and OFB modes.
  • Know why linear ciphers are weak and how confusion/diffusion strengthen ciphers.
  • Recall key sizes and block sizes for DES and AES.
  • Be aware of practical attacks (e.g., side-channel) and how to defend against them.
