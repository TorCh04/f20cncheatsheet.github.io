Network Security Exam Revision: VPNs, TLS, and IPSec

1. Virtual Private Networks (VPNs)

  • Definition: A VPN is a security technology that enables authorized users to use an unsecured public network (like the Internet) as if it were a secure private network.
  • How it works: Encrypts all data transmitted between the remote endpoint and the network.
  • VPN Tunnel: Connects two devices securely over an untrusted network.
  • Types of VPNs:
    • Remote access VPN: User-to-LAN connection for remote users.
    • Site-to-site VPN: Connects multiple sites over the Internet.
  • Always-on VPNs: Stay connected at all times, no need to reconnect each session.
  • Main secure protocols:
    • Transport Layer Security (TLS)
    • IP Security (IPSec)

2. TLS VPN

  • Transport Layer Security (TLS): Provides a high degree of protection for VPNs.
  • Standard for secure communications (including VPNs).
  • Clientless VPN: Users can connect via a browser without a separate VPN client.

3. IPSec VPN

  • IP Security (IPSec): A suite of protocols for securing IP communications.
  • How it works: Encrypts and authenticates each IP packet in a session.
  • Encryption Modes:
    • Transport Mode: Encrypts only the data (payload), header remains unencrypted.
    • Tunnel Mode: Encrypts both header and data.
  • Mechanism: Adds new headers to the IP packet; the original packet becomes the data portion of the new packet.
  • Robustness: More robust than TLS because it secures IP, the foundation of all TCP/IP protocols.

4. IPSec vs TLS

Feature IPSec TLS
Layer Network Layer Application Layer
Encryption Scope Entire IP packet (Tunnel mode), or just payload (Transport mode) Application data (e.g., web traffic)
Client Requirement Usually requires VPN client Can be clientless (browser-based)
Use Cases Site-to-site, remote access Remote access, web-based access
Robustness More robust (protects all IP traffic) Less robust (protects specific applications)
