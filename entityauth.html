1. Overview of Security Properties

  • Confidentiality: Encryption (historical, symmetric, asymmetric), hybrid encryption (asymmetric for key exchange, symmetric for data).
  • Integrity: Hash functions, MACs, digital signatures (also provide non-repudiation).
  • Access Control: Firewalls.
  • Attack Detection: Intrusion detection systems.

2. Authentication Basics

  • Authentication: Confirming someone/something is authentic (entity or user authentication).
  • Parties: Claimant (A) and Verifier (B).
  • Passes: One-pass (single message) or multi-pass (challenge-response).
  • Factors: Can use multiple independent pieces of information.

3. Types of Authentication Factors

  • Something you know: Password, PIN.
  • Something you have: Token, card, phone, private key.
  • Something you are: Biometric (fingerprint, face, iris, etc.).
  • Somewhere you are: Location-based (e.g., IP address).
  • Someone you know: Social authentication.
Best security: Use factors from different categories (e.g., password + card).

4. Authentication Lifecycle

  • Registration/Issuance: Setting up authentication info.
  • Login/Authentication: Using info to authenticate.
  • Maintenance/Recovery: Updating or recovering info.
Security: Confidentiality, integrity, and availability of authentication info.
Usability: Convenience and memorability affect security.

5. Usability & Security of Factors

Biometrics ("Something you are")
  • Issuance: No need to choose info, but must record biometric securely.
  • Login: Requires correct interaction; must detect liveness.
  • Maintenance: Limited options if compromised or unusable.
Tokens/Cards ("Something you have")
  • Issuance: Securely share device/info.
  • Login: Must possess and use device correctly.
  • Maintenance: Re-issuance if lost/damaged.
Passwords ("Something you know")
  • Issuance: Must choose strong, memorable password.
  • Login: 100% recall required; must protect entry.
  • Maintenance: Avoid overlap with old passwords.

6. Multi-Factor Authentication

Using two knowledge-based factors (e.g., PIN + password) is better than one, but not as secure as using factors from different categories.
  • Multiplicative security: If both must be guessed together: Work = W(PIN) Ã— W(password)
  • Additive security: If errors are reported separately: Work = W(PIN) + W(password)

7. Biometrics

  • Types: Physiological (fingerprint, iris, face), Behavioural (voice, typing pattern).
  • Applications: Identification (one-to-many), Verification (one-to-one).
  • Key Considerations: Usability (enrolment, login), Security (false acceptance/rejection), Privacy (template storage).
OutcomeDescriptionResultImpact
True Positive (TP)Legitimate user acceptedCorrectGood usability
False Negative (FN)Legitimate user rejectedIncorrectBad usability
True Negative (TN)Attacker rejectedCorrectGood security
False Positive (FP)Attacker acceptedIncorrectBad security
  • False Acceptance Rate (FAR): FAR = FP / (FP + TN) (should be < 0.1%)
  • False Rejection Rate (FRR): FRR = FN / (FN + TP)
  • Equal Error Rate (EER): Point where FAR = FRR (lower is better).

8. Password Security

  • Attacks: Shoulder surfing, social engineering, server impersonation, client compromise, eavesdropping, user impersonation, DoS, database compromise.
  • Mitigations: Masking input, user education, anti-malware, encryption, limiting attempts, hashing and salting passwords.
Passwords should be stored as salted hashes, not plain text.
  • Hashing: Store h(password) instead of password.
  • Salting: Store h(salt, password) with unique salt per user.
  • Salting effect: Prevents attackers from using precomputed hash tables for all users.

9. Dynamic Passwords & One-Time Passwords

  • One-time password lists: Each password used only once; cumbersome to manage.
  • Lamport Hash Chains: Use iterated hashes for one-time passwords without storing a list.
  • Variable Passwords: Use nonces (random, timestamp, sequence) to make each authentication unique.

10. "Something You Have" Authentication

  • Tokens: Hardware (e.g., OTP display), Software (e.g., phone app).
  • Security Keys: USB, Lightning, or NFC dongles with cryptographic info.
  • Smart Cards: Physical cards with embedded chips, often require a reader.

11. "Something You Are" Authentication

  • Biometrics: Used for identification (one-to-many) and authentication (one-to-one).
  • Qualities: Uniqueness, universality, permanence, measurability, acceptability.
  • System Architecture: Data capture, signal processing, matching, storage, decision.

12. Physiological & Behavioural Biometrics

  • Physiological: Fingerprint, face, retina, iris, palm, voice.
  • Behavioural: Keystroke dynamics, signature, gait.
Keystroke dynamics: Measures dwell and flight time; no special hardware needed.

13. Error Ratings in Biometrics

  • False Rejection Rate (FRR): Legitimate user rejected. FRR = FN / (FN + TP)
  • False Acceptance Rate (FAR): Attacker accepted. FAR = FP / (FP + TN)
  • Goal: Minimize both FRR and FAR (trade-off exists).

15. Key Exam Tips

  • Understand the difference between authentication factors and why combining different types increases security.
  • Be able to explain how password hashing and salting work, and why they are important.
  • Know the main types of biometric errors (FAR, FRR) and their implications.
  • Be familiar with common attacks on authentication systems and their mitigations.
  • Practice drawing and explaining authentication protocols (e.g., challenge-response, Lamport hash chains).
