Network Security: Hash Functions & Data Integrity

1. Hash Functions

  • A hash function maps input of arbitrary size to a fixed-size output.
  • Cryptographic hash functions have special security properties (see below).
  • Notation: h(x) is the hash of input x.
  • The output is called a hash, hash value, or message digest.

2. Properties of Cryptographic Hash Functions

  • One-way (Preimage resistance): Given h(x), it is hard to find x.
  • Weak collision resistance (Second preimage resistance): Given x and h(x), hard to find y â‰  x such that h(x) = h(y).
  • Strong collision resistance: Hard to find any x and y such that h(x) = h(y).
Examples:
  • Common hash functions: MD5, SHA-1, SHA-256, SHA3-256
  • MD5 and SHA-1 are not secure (not collision resistant).

3. Applications of Hash Functions

  • Password storage:
    • Passwords are hashed and stored.
    • One-way property protects against leaks.
    • Password hash functions (e.g., bcrypt, scrypt) must be slow to prevent guessing attacks.
  • Key derivation:
    • Derive encryption keys from passphrases (e.g., pbkdf2, Argon2).
  • Proof-of-work:
    • Used in cryptocurrencies; relies on one-way property.
  • Data integrity:
    • Weak collision resistance is important.
  • Digital signatures:
    • Collision resistance ensures non-repudiation.

4. Confidentiality vs. Integrity

  • Ciphers (e.g., AES, ChaCha20) provide confidentiality but not integrity.
  • Modification of ciphertext can alter plaintext undetectably.
  • Solution: Use a hash function to verify integrity.
Integrity with Hash Functions:
  1. Alice sends x over an insecure channel.
  2. Alice sends h(x) over an authentic channel (e.g., phone, trusted website).
  3. Bob receives x', computes h(x'), and checks if it matches h(x).
  4. If they match, x' = x (by weak collision resistance).

5. Message Authentication Codes (MACs)

  • Hash functions alone require a secure channel for h(x).
  • MACs (e.g., HMAC) use a secret key: hK(x).
  • HMACs are built from cryptographic hash functions (e.g., HMAC-SHA256).
  • Both sender and receiver share the key K (symmetric).
How HMAC Works:
  1. Alice sends x and hK(x) to Bob.
  2. Bob receives x' and h', computes hK(x'), and checks if it matches h'.
