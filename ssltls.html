SSL/TLS & Web Security Protocols - Exam Revision

1. Web Security Protocols

  • Encryption protects data in transit from interception.
  • Authentication verifies website identity (prevents imposters).
  • TLS (Transport Layer Security) is the standard protocol, replacing SSL (Secure Sockets Layer).
  • Secure sites use https:// (not http://).
What is the predecessor to TLS?
SSL (Secure Sockets Layer).
What does TLS secure?
HTTP, IMAP, and other TCP/IP protocols.
What must be configured for TLS/SSL to work?
Both browser and web server must be properly configured.

2. TLS/SSL Overview

  • Browser requests server authentication (identity check).
  • Authentication uses certificates from trusted Certificate Authorities (CAs).
  • TLS encrypts data and checks for tampering.
  • Essential for confidential data (e.g., credit cards).

History

  • SSL developed by Netscape (1994), SSL 2.0 (1995), SSL 3.0 (1996).
  • TLS introduced by IETF in 1999 (based on SSL 3.0).
  • TLS 1.1 (2006), TLS 1.2 (2008), TLS 1.3 (2018).
  • SSL 2.0 and 3.0 are deprecated due to security flaws.

Enhancements in TLS

  • Uses HMAC instead of SSL MAC.
  • More alert messages.
  • Certificates can use intermediary authorities.

3. Cipher Suites

  • Define how SSL/TLS secures a connection.
  • Components:
    • Key Exchange Algorithm (e.g., RSA, DH, ECDHE)
    • Authentication/Digital Signature (e.g., RSA, ECDSA)
    • Bulk Encryption (e.g., AES, CHACHA20)
    • MAC Algorithm (e.g., SHA-256, POLY1305)
  • Example: TLS_DH_DSA_AES-128-ECB_SHA256

4. Client-Server Communication

  • Client and server agree on protocol version and cipher suite.
  • Authenticate each other using digital certificates.
  • Use asymmetric encryption to generate a shared secret key.
  • Shared key used for symmetric encryption of messages.

5. Notable Attacks

POODLE Attack (2014)

  • Exploits SSL 3.0 fallback for legacy support.
  • Attacker can force downgrade and exploit SSL 3.0 vulnerabilities.

Heartbleed (2014)

  • Found in OpenSSL's heartbeat extension.
  • Allows attacker to read server memory by sending malformed heartbeat requests.

TLS 1.3 Vulnerability (2019)

  • Cache attacks found in several TLS 1.3 implementations.
  • BearSSL and BoringSSL not vulnerable; others (OpenSSL, Apple CoreTLS, etc.) were.
  • RSA is the only known downgrade attack on TLS 1.3.

6. Benefits of TLS/SSL

  • Strong authentication, privacy, and integrity.
  • Protection against masquerade, MiTM, and replay attacks.
  • Interoperable with most browsers, OS, and servers.
  • Flexible algorithms and easy deployment/use.

7. Limitations of TLS/SSL

  • Increased processor load (especially during connection setup).
  • Administrative overhead (certificate management, configuration).

8. Example Scenario: E-Commerce

  • Client provides credit card info to shopping site.
  • Protocol first validates server certificate.
  • Credit card info sent as ciphertext.
  • Only server is authenticated (if certificate is from trusted CA).
  • TLS/SSL must be enabled on order form pages.
