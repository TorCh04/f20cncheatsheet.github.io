Network Security Exam Revision: Firewalls

1. What is a Firewall?

  • First line of defence in a network.
  • Monitors and controls incoming/outgoing network traffic based on security rules.
  • Positioned between a local network and the Internet, filtering harmful traffic.
  • Can be hardware, software, or both.

2. Firewall Functions

  • Limits the spread of malware.
  • Examines both outgoing and incoming packets (bidirectional).
  • Allows approved packets; blocks or denies suspicious packets based on rules.
  • Rule-based firewalls use specific criteria to accept or deny packets.

3. Firewall Rules

  • Parameters include:
    • Source/Destination Address (IP, range, MAC, hostname)
    • Source/Destination Port (specific or range)
    • Protocol (e.g., TCP, UDP)
    • Direction (Incoming, Outgoing, Both)
    • Priority (order of rule application)
    • Time (active schedule)
    • Context (e.g., on-site vs remote)
    • Action (allow, deny, etc.)

Examples of Firewall Rule Actions

  • Allow inbound SMTP (email) connections.
  • Allow outbound SMTP connections.
  • Default rule applies if no previous rule matches.

Firewall Rules Configuration Examples

  • Block all connections from a specific IP:
    sudo ufw deny from 15.15.15.51
  • Allow all incoming HTTP traffic:
    sudo ufw allow http

Remarks

  • Older firewalls processed rules sequentially (IF-THEN-ELSE); order was crucial.
  • Modern firewalls use priorities to resolve rule conflicts.

More Flexible Firewalls

  • Policy-based firewalls allow generic statements (e.g., allow all traffic from a subnet to a port).
  • Can include content/URL filtering (e.g., block specific websites).

4. Firewall Categories

Term/Concept Definition
Hardware Firewall Separate device; more features, costlier, harder to manage.
Software Firewall Runs on a device; can be compromised if host is infected.
Host-based Firewall Protects a single endpoint; included in modern OSes.
Appliance Firewall Hardware device for entire network protection.
Virtual Firewall Runs in the cloud; used when hardware is impractical.
Stateless Packet Filtering Examines each packet individually (Layer 3).
Stateful Packet Filtering Tracks connection state (Layer 4); allows/denies based on session.
Access Control List (ACL) Set of rules to permit/deny traffic; not a device.
Policy-based Firewall Uses generic policies; can include content/URL filtering.

5. Specialised Firewalls

Web Application Firewall (WAF)

  • Examines HTTP applications.
  • Blocks attacks like XSS and SQL injection.
  • Can be hardware or software.

Next-Generation Firewall (NGFW)

  • Provides features beyond traditional firewalls.
  • Uses deep packet inspection for malware detection.
  • Includes URL filtering and intrusion prevention.

Unified Threat Management (UTM)

  • All-in-one device: packet filtering, antispam, antiphishing, antispyware, encryption, intrusion protection, web filtering.
  • Drawback: Single point of failure (SPOF).

6. Final Remarks

  • Firewalls are often seen as the ultimate security device, but this is inaccurate.
  • They have limitations but remain a key element in network security.
