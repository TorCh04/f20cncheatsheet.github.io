2. Goals of Cryptography

  • Confidentiality: Preventing disclosure of information to unauthorized entities
  • Integrity: Preventing unauthorized modification of information
  • Authenticity: Preventing attribution of data to false origin

The aim is to ensure these properties even when messages can be eavesdropped, intercepted, forged, or modified.

3. Adversary Model (Threat Model)

  • Passive Adversary: Can eavesdrop on all messages (read-only)
  • Active Adversary: Can forge and modify messages (read/write)
Symmetric Encryption:
Alice encrypts plaintext m with key k: c = Ek(m)
Bob decrypts c with key k: m = Dk(c) = Ek-1(c)

4. Attacks Against Cryptosystems

  • Ciphertext-only attack: Attacker knows only ciphertexts
  • Known-plaintext attack: Attacker knows plaintext-ciphertext pairs
  • Chosen-plaintext attack: Attacker can choose plaintexts and get ciphertexts
  • Chosen-ciphertext attack: Attacker can choose ciphertexts and get plaintexts
Kerckhoff's Principle: The security of a cryptosystem should depend only on the secrecy of the key, not the algorithm.

6. Symmetric Cryptography Systems

  • Transposition: Permute symbols in the block
  • Substitution: Replace symbols with others

6.1 Simple Transposition Cipher

  • Permutes symbols in a block using a key (permutation)
  • Encryption: Ee(m) = (me(1), me(2), ..., me(t))
  • Decryption: Apply the inverse permutation
Example: Permutation e = (4,3,2,6,1,5) encrypts secrete as rcetse

6.3 Cryptanalysis of Transposition Ciphers

  • Preserves letter frequencies
  • Cryptanalyst uses n-gram frequencies and tries reverse transpositions

7. Simple (Monoalphabetic Substitution) Ciphers

  • Each letter is replaced by another letter using a permutation of the alphabet
  • Encryption: Ee(m) = e(m1)e(m2)...e(mt)
  • Decryption: De(c) = e-1(c1)...e-1(ct)

7.1 Caesar Cipher

  • Shift each letter by a fixed amount (key e)
  • Example: e = 3, A → D, B → E, etc.
Key Mapping:
Plain: ABCDEFGHIJKLMNOPQRSTUVWXYZ
Cipher: DEFGHIJKLMNOPQRSTUVWXYZABC
Example:
Plaintext: the quick brown fox jumps over the lazy dog
Ciphertext: WKH TXLFN EURZQ IRA MXPSV RYHU WKH ODCB GRJ
Encryption: Ee(x) = x + e mod 26
Decryption: De(x) = x - e mod 26

8. Breaking Caesar's Cipher

  • Brute Force Attack: Try all 26 keys and pick the most plausible plaintext
  • Letter Frequency Attack: Compare letter frequencies in ciphertext to typical plaintext
Caesar Cipher is weak: Only 26 possible keys; easily broken with passive, ciphertext-only attacks.

9. Polyalphabetic Substitution (Vigenère Cipher)

  • Uses a keyword to apply different shifts to each letter
  • Flattens frequency distribution, making analysis harder
  • Vigenère Square helps with encryption/decryption

9.1 Attacking Vigenère Cipher

  • If key length is known, group ciphertext accordingly and analyze each group
  • Guess key length by looking for repeated patterns (Kasiski examination)
  • Statistical methods (e.g., Measure of Roughness) can help confirm key length
MR = Σi=0n-1 (pi - 1/n)2 = Σi=0n-1 pi2 - 1/n
For English, MR ≈ 0.028. Use this to verify key length guesses.

10. One-Time Pads (Vernam Cipher)

  • Key is random, as long as the plaintext, and used only once
  • Encryption (binary): c = m ⊕ k
  • Unconditionally secure if keys are random and never reused
  • Impractical due to key distribution and management
  • Reusing keys makes the cipher weak (Running Key cipher)
  • Pseudorandom sequences (stream ciphers) are used in practice, but are only computationally secure
Why unconditionally secure?
Given a ciphertext, any plaintext guess yields a possible key, so attacker cannot determine the correct plaintext.
