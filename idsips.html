1. Intrusion

Intrusion: A sequence of related actions by a malicious adversary that compromises a target system.

  • Guessing/cracking passwords
  • Copying sensitive databases (e.g., credit card numbers)
  • Viewing sensitive data
  • Running packet sniffers to capture credentials
  • Using unattended, logged-in devices without permission

2. Intrusion Detection System (IDS)

IDS: Monitors networks/systems for malicious activities or policy violations.

  • Implemented as a device or software application
  • Detects attacks or potential attacks
  • Produces reports/alerts for administrators
  • Cannot usually stop attacks (does not block packets)

2.1 IDS Principles

  • Assumes intruder behavior differs from legitimate users
  • Patterns of legitimate behavior established from history
  • No exact distinction between attack and normal activity
  • Loose interpretation: More intruders caught, more false positives
    Tight interpretation: More false negatives (missed intrusions)
  • False Positive: Legitimate activity flagged as intrusion
    False Negative: Malicious activity not detected

2.2 IDS Confusion Matrix

Actual Intrusion Actual Normal
IDS: Intrusion True Positive False Positive
IDS: Normal False Negative True Negative

2.3 IDS Types

  • Host-based IDS (HIDS): Monitors a specific host; expensive to deploy everywhere; used for critical servers.
  • Network-based IDS (NIDS): Monitors network traffic; cheaper; monitors at network level.

2.4 IDS Detection Styles

  • Signature-based Detection:
    • Uses known attack patterns (signatures)
    • Detects only known attacks (cannot detect new attacks)
    • Example: Regex union\s+select for SQL injection
  • Anomaly-based Detection:
    • Learns normal network behavior over time
    • Uses ML/statistics to detect deviations
    • Can detect new attacks, but may miss attacks similar to normal activity

3. Intrusion Prevention System (IPS)

  • Can block connections, alert, log, quarantine hosts, etc.
  • Types: Host-based (HIPS) and Network-based (NIPS)

3.1 IDS vs IPS

  • IDS does not slow down networks; IPS can
  • IPS false positives can block legitimate traffic; IDS only alerts/logs

3.2 Combining IDS and IPS

  • Some solutions combine both IDS and IPS
  • Start with IDS mode to observe, then enable IPS for active protection
Exam Tips:
  • Know definitions and differences between IDS and IPS
  • Understand types (HIDS/NIDS, HIPS/NIPS) and detection methods
  • Be able to explain false positives/negatives and their impact
  • Give examples of signature and anomaly-based detection
