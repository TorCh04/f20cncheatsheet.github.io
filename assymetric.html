1. Symmetric vs Asymmetric Cryptosystems

  • Symmetric (Secret-Key): Same key for encryption and decryption (e.g., AES).
  • Asymmetric (Public-Key): Two keys: public (encryption) and private (decryption). E.g., RSA, ElGamal.

2. Limitations of Symmetric Encryption

  • Key distribution is difficult for multiple recipients.
  • Secure key exchange is a challenge.

3. Public-Key Encryption

  • Each user has a public and private key.
  • Encrypt with recipient's public key; decrypt with their private key.
  • Security relies on infeasibility of deriving private key from public key.

4. RSA Cryptosystem

Key Generation

  1. Choose large primes p and q; compute n = p * q.
  2. Choose e such that gcd(e, (p-1)(q-1)) = 1.
  3. Compute d = e-1 mod (p-1)(q-1).
  4. Public key: (e, n); Private key: d.

Encryption

C = Me mod n

Decryption

M = Cd mod n
Note: Both encryption and decryption are modular exponentiations.

Security

  • Based on difficulty of factoring large integers.
  • RSA keys must be much larger than symmetric keys (â‰¥2048 bits recommended).

5. Hybrid Encryption

  • Symmetric encryption for data (fast).
  • Asymmetric encryption for exchanging the symmetric key (secure key distribution).

6. ElGamal Cryptosystem

  • Public-key cryptosystem using randomization (non-deterministic).

Key Generation

  1. Choose prime p, generator g.
  2. Choose private key x; compute y = gx mod p.
  3. Public key: (p, g, y); Private key: x.

Encryption

(a, b) = (gk mod p, m * yk mod p)

Decryption

m = b * (ax)-1 mod p
Security: Based on the discrete logarithm problem.

7. Diffie-Hellman Key Exchange

Purpose: Establish a shared secret over an insecure channel.
  1. Alice: picks random x, sends X = gx mod p.
  2. Bob: picks random y, sends Y = gy mod p.
  3. Both compute shared key: K = gxy mod p.
Security: Based on the difficulty of the discrete logarithm problem.

8. Security Problems & Attacks

  • Man-in-the-Middle Attack: Active attacker intercepts and alters key exchange.
  • RSA, ElGamal, DH provide confidentiality, but not integrity or authentication by themselves.

9. Digital Signatures & Certificates

  • Sign with private key; verify with public key.
  • Provides authenticity and integrity.
  • Public keys are distributed in digital certificates signed by Certificate Authorities (CAs).

Summary Table

  • Symmetric: Fast, same key, e.g., AES.
  • Asymmetric: Easier key management, e.g., RSA, ElGamal.
  • Hybrid: Combines both for secure, efficient communication.
  • Integrity: Provided by hashes, MACs, or digital signatures.
  • Authentication: Requires digital signatures and certificates.

Key Problems in Asymmetric Cryptography

  • Factoring: Find prime factors of n.
  • RSA Inversion: Given c, find m such that me = c mod n.
  • Discrete Logarithm: Given g, p, a, find x such that gx = a mod p.
  • Diffie-Hellman: Given gx and gy, compute gxy mod p.
